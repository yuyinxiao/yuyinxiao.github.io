hexo blog + github + stackEdit

基于hexo搭建blog；source文件夹下为markdown源文件；现用stackEdit编辑，可直接同步到github master分支

pipeline

！！切记，内容不要包含敏感字段等。

  1. stackEdit绑定github repo和文件夹，对应为hexo的source文件夹；
  2. 复制模版编写文章，注意md的格式，包含title，tags，date，categoties等；
  3. stackEdit同步到github；
  4. hexo d -g部署与发布即可。

hexo cli

  1. hexo n == hexo new
  2. hexo g == hexo generate
  3. hexo s == hexo server
  4. hexo d == hexo deploy
  5. hexo s -g 生成并本地预览
  6. hexo d -g 生成并上传
